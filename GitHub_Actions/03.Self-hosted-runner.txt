GitHub Actions Self-Hosted Runner Setup on AWS EC2
--------------------------------------------------

Step 1: Launch EC2 Instance
- Use Ubuntu 22.04 LTS (free tier eligible)
- Instance Type: t2.micro or larger
- Security Group: Allow SSH (port 22) and outbound HTTPS (port 443)

Step 2: Connect to EC2 via SSH
ssh -i "your-key.pem" ubuntu@ec2-ip-address

Step 3: Install Dependencies
sudo apt update -y
sudo apt install -y curl unzip

Step 4: Download & Configure Runner
1. Create directory and download:
mkdir actions-runner && cd actions-runner
curl -o actions-runner-linux-x64-2.311.0.tar.gz -L https://github.com/actions/runner/releases/download/v2.311.0/actions-runner-linux-x64-2.311.0.tar.gz
tar xzf actions-runner-linux-x64-2.311.0.tar.gz

2. Configure runner:
./config.sh --url https://github.com/your-username/your-repo --token YOUR_RUNNER_TOKEN
(Get token from repo Settings -> Actions -> Runners -> New self-hosted runner)

Step 5: Install as Service
sudo ./svc.sh install
sudo ./svc.sh start

Step 6: Verify in GitHub
- Check repo Settings -> Actions -> Runners
- Should see runner with "Idle" status

Important Notes:
- Runner directory: /home/ubuntu/actions-runner
- Logs location: ./diag/Runner_*.log
- Restart runner: sudo ./svc.sh restart
- Remove runner: ./config.sh remove --token YOUR_TOKEN
- Runner auto-updates by default

Troubleshooting:
- Check runner logs if jobs not picking up
- Verify network connectivity to GitHub
- Ensure token is correct and not expired
